# Bugfix Requirements Document

## Introduction

The BeamLab Ultimate web application has two related branding and UI consistency bugs. First, `BEAMLAB_COMPANY.name` is set to `'BeamLab'` instead of `'BeamLab Ultimate'`, causing the wrong product name to appear on PDF report cover pages, headers, and footers. Second, `ReportsPage.tsx` hardcodes the project name as `"BeamLab Project"` in both the cover page title and the PDF export handler, ignoring the actual project name from the store. Additionally, several pages and layout components do not consistently apply the established design system (logo, brand colors, design tokens), resulting in bare or unpolished UI surfaces.

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN any PDF report is generated THEN the system displays "BeamLab" as the product name on the cover page, headers, and footers instead of "BeamLab Ultimate"

1.2 WHEN the ReportsPage cover page is rendered THEN the system displays the hardcoded title "BeamLab Project" instead of the actual project name from the project store

1.3 WHEN the user triggers PDF export from ReportsPage THEN the system passes the hardcoded string `'BeamLab Project'` as the project name to the PDF generator instead of reading the actual project name from the store

1.4 WHEN pages and layout components (Dashboard, AppShell, ModernWorkspace, WorkspaceLayout, navigation components) are rendered THEN the system displays dark backgrounds without the BeamLab Ultimate logo or consistent brand color usage from the design token system

### Expected Behavior (Correct)

2.1 WHEN any PDF report is generated THEN the system SHALL display "BeamLab Ultimate" as the product name on the cover page, headers, and footers by reading from `BEAMLAB_COMPANY.name`

2.2 WHEN the ReportsPage cover page is rendered THEN the system SHALL display the actual project name sourced from the project store

2.3 WHEN the user triggers PDF export from ReportsPage THEN the system SHALL pass the actual project name from the project store to the PDF generator

2.4 WHEN pages and layout components are rendered THEN the system SHALL consistently display the BeamLab Ultimate logo and apply brand colors and design tokens from the established design system

### Unchanged Behavior (Regression Prevention)

3.1 WHEN `BEAMLAB_COMPANY.generatedByLine` is used THEN the system SHALL CONTINUE TO display "Generated by BeamLab Ultimate — Professional Structural Engineering Software" unchanged

3.2 WHEN the Logo component is rendered with any supported variant (full, icon, wordmark) or size THEN the system SHALL CONTINUE TO render the correct SVG assets with proper accessibility attributes

3.3 WHEN PDF reports are generated with a valid non-empty project name THEN the system SHALL CONTINUE TO produce correctly structured A4 documents with all existing sections (cover page, TOC, executive summary, design basis, results)

3.4 WHEN pages that already correctly use the design system are rendered THEN the system SHALL CONTINUE TO display their existing branding and layout without regression

3.5 WHEN `BEAMLAB_COLORS`, `BEAMLAB_COLORS_RGB`, and `BEAMLAB_LOGO` constants are consumed by any service or component THEN the system SHALL CONTINUE TO provide the same color values and logo paths unchanged
